## Recommendation

**Merge the branch — it is a net improvement over `main` with no behavioral
regressions — but carry five follow-ups, two of which should land before any
council runs in earnest.** The branch (1 commit, `SKILL.md +144/-11`, plus a
PLAN.md clarifier and an example back-link) pins the record file format, the
memory topic format, a deterministic topic-naming rule, the scratchpad header,
and a two-gate post-synthesis conformance check. The spec-tightening is sound,
internally consistent, and **dogfooded**: every pinned template matches its cited
worked example, and the prior session's own record + memory file both conform and
cross-link bidirectionally — the format passes its own gates.

The review found no defects in shipped behavior, but it found that the pinned
formats and the conformance check are **incomplete and not yet mechanically
verifiable** — and the branch shipped its own counterexamples that prove it.
Priority-ordered follow-ups:

1. **Hole A — back-link cardinality (latent data-loss bug; before real use).**
   The memory template pins exactly one `→ record:` line, but the topic-naming
   rule says updates "keep history and add the new decision." Contradiction fires
   the second time any council touches a topic: session B overwrites session A's
   back-link, A's already-merged record's `→ memory updated:` dangles, and Gate 2
   silently breaks retroactively with nobody re-running it. Decide now whether a
   topic carries a back-link *history* (list) or *latest-only*.
2. **Archive, don't delete, the scratchpad (before real use).** Both verbs say
   "run the conformance check, then **remove the scratchpad**" — destroying the
   evidence Gate 1 (dissents-preserved) self-certifies against. Move
   `scratch/<id>.md` to e.g. `records/<id>.scratch.md` instead. Smaller than any
   gate (it's deleting a deletion); makes Gate 1 and Hole A retro-auditable.
3. **Make Gate 2 mechanical (one enum constraint).** Require `→ record:` to be
   either a `records/<id>.md` path or a single reserved literal (e.g.
   `STANDING`) — no free prose. Converts our strongest gate from human-skim to a
   ~15-line grep. Currently blocked by exactly one shipped counterexample
   (`testing-standards.md`'s prose back-link).
4. **Pin the empty-topic and multi-topic `→ memory updated:` forms.** A meeting
   that decides nothing durable (zero topics) and a session that writes 3 topics
   are both common; the template only shows the single-topic form. Finish the
   templates so chairs don't improvise.
5. **F1 + F2 (cheap honesty fixes).** F1: the record template says follow-up
   owners are seat names, but the cited example uses short forms (`owner: qa` vs
   `qa-engineer`) — fix the example or loosen the rule. F2: add one sentence to
   the *prior* record's reasoning trail noting Phase 2's meeting-verb scope was
   already on `main`; what shipped was the unpinned format tail.

## Reasoning trail

- **The spec is dogfooded, which is the strongest possible evidence it's real.**
  The chair verified each pinned template against its cited example and against
  the prior session's own record/memory — all conform and cross-link both ways.
  A format that passes its own two gates is not aspirational. This is why the
  recommendation is *merge*, not *rework*.
- **F2 (the relabel) is real but ranks low.** `git show main:` confirms the
  `## Verb: meeting` round-loop and synthesis→record existed on `main` before
  this branch; Phase 2's headline deliverable predated the session. The prior
  record's "Phase 2 turned out not to be new code" is honest about *the branch*
  but quietly redefines *the phase* to match what got done. The chair and QA
  flagged it as record-honesty; the PM resolved its severity from the outcome
  lens: **the user who runs `/council` consumes behavior, not the phase label**,
  and behavior improved. So F2 is a one-sentence honesty fix, not a reopen.
- **The conformance check is the right *shape* but the wrong *completeness*.**
  Last session trimmed QA's ~11-item checklist to two gates (PM won on scope).
  This review confirms two gates was the right call on *shape* — but found the
  two surviving gates under-specified, not over-trimmed: Gate 1 is self-reported
  by the chair against evidence it then deletes (unauditable by construction),
  and Gate 2 — the genuinely machine-checkable one — isn't runnable end-to-end
  because the branch shipped a free-prose back-link that breaks any closure
  script. The fixes are narrow (enum + template completeness + archive), not a
  re-expanded checklist.
- **The QA↔PM tension *converged* this session.** Unlike the prior session
  (where the chair adjudicated against QA), here the PM independently conceded
  all three of QA's re-aimed asks on the merits — because they cleared the PM's
  own bar (cheaper-than-human detection, finishing the templates the PM leaned
  on, a real latent bug). What remains is not a disagreement about *whether*, but
  a preserved guardrail about *shape*. See Dissents.
- **Metric for Phase 2 success (named by PM):** run two independent councils and
  read both records cold — success iff both records are structurally identical in
  shape and the memory directory has one file per durable subject with zero
  near-duplicates and zero dangling cross-links. Hole A threatens the second
  clause; the Gate-2 enum fix is what lets you *check* it.

## Dissents (preserved)

This review was notable for **convergence rather than conflict** — recording that
honestly per Gate 1 (the section is not empty for empty's sake; the genuine
disagreement is narrow and below).

**QA-engineer's re-aimed standing dissent, and its partial resolution.**

- **QA's position:** the evidence partially *reopens* the prior session's
  preserved dissent — but NOT by restoring the ~11-checkbox list. The two gates
  the council kept are not yet verifiable or cardinality-complete, and this
  branch shipped the proof (the prose back-link that breaks Gate 2; the
  one-`→ record:`-slot template that loses history on the second session). QA
  wants: Gate 2 made mechanical (enum), the empty/multi-topic forms pinned, and
  Hole A's cardinality resolved — plus flagged that Gate 1 is unauditable once
  the scratchpad is deleted.
- **PM's response — conceded on substance, held on shape.** The PM conceded all
  three re-aimed asks on the merits (each clears the PM's own bar), explicitly
  noting the line moved because the branch *shipped the counterexample*, not
  because of argument. The PM held one line: these land as **template/spec fixes
  plus at most one tightened Gate-2 clause — NO third gate.** "Two gates, not
  eleven" still stands; the PM will re-dissent if a Gate 3 is smuggled in under
  cover of these fixes. The PM added its own follow-up — archive-don't-delete the
  scratchpad — as the cheapest *faithfulness* companion to its legibility metric.
- **Resolution (chair):** adopt all five follow-ups; honor the PM's shape
  constraint — Hole A, empty/multi-topic, and the Gate-2 enum are
  **template/spec completions and a tightening of the existing Gate 2**, not a
  new gate. The live tension to carry forward is unchanged in axis but narrowed
  in scope: **QA optimizes against format drift and un-auditability; PM optimizes
  against process overhead.** This session they agreed on *what* to fix; the only
  residual is *how it's packaged* — and the standing rule is: completeness fixes
  go in the templates, verification stays at two gates.

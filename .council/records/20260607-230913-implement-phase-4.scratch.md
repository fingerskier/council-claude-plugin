## Turn 2 — qa-engineer

Read SKILL.md `work` steps 2-6, "Spawning a seat", scratch-header example, the `work_budget` blocks. The chair's mechanism is fireable — but only with one fix.

**Crux fix:** the existing `**Started:** <YYYY-MM-DD HH:MM>` header field is a **trap, not a timer** — minute-granular, human-formatted, locale-ambiguous. Subtracting those strings is exactly the brittle arithmetic that breeds a phantom. Persist a separate machine epoch or this is no more real than Phase-3 wall-clock.

### Mechanism (executable by an LLM-orchestrator, which holds no live timer)
- **Record start (work step 2/3):** run `date +%s` once at session open, write into scratchpad header as a new pinned field under `**Started:**`: `- **Started (epoch):** 1749337740`. The epoch is the single source of truth for elapsed time; never derive it from the `HH:MM` string (no date-rollover handling).
- **Check at each turn boundary (step 6c):** at the same checkpoint where the chair counts turns and `wc -c`s the scratchpad, run `date +%s` for `now`; fire iff `max_wall_seconds` set and `now - started_epoch ≥ max_wall_seconds`.
- **Fireable, not phantom:** every input is a concrete shell read (`date +%s` both ends, epoch persisted in the re-read artifact, integer subtraction) — same cadence/exactness as `scratch_max_bytes`(`wc -c`)/`max_turns`. That parity IS the argument it's real.
- **Honest caveat (must be in prose):** checked at turn boundaries → bounds when the *next* turn starts, not a mid-turn interrupt; a turn in flight finishes first, so total time can overshoot by one turn's duration. Same looseness shape as `max_turns`.

### Edge cases (highest-risk first)
- **E2/E5 (top severity): `max_wall_seconds: 0`, negative, or field-present-but-no-epoch-line → must be UNARMED**, not instant-fire. A literal `now - start ≥ 0` fires before turn 1 → a zero-turn run that declares "done". Pin: **absent | 0 | negative → unarmed**.
- E1 absent → unarmed (only other four bound the run). E4 single long turn → fires at next boundary (overshoot), stop reason correctly `max_wall_seconds` not `max_turns`. E6 first-to-fire tie-break = **step-6c bullet order** (chair-done > max_turns > scratch-size > wall-seconds > user-stop) for a reproducible recorded reason. E7 use `date +%s` (UTC epoch) → immune to DST/locale, which is *why* we don't reuse the `HH:MM` string.

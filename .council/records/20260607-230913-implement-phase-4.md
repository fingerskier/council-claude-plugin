## Reasoning trail

- **Seat selection.** Chose {staff-engineer, qa-engineer, product-manager}; excluded security-engineer deliberately. This is spec/doc work on a budget knob: no untrusted input, no secrets, no auth boundary, no new attack surface, and worktree isolation is untouched. A security seat would have been ceremony.
- **QA's mechanism crux.** The fireable contract hinges on one finding: you cannot do wall-clock arithmetic against the minute-granular `**Started:**` header. So QA specified a separate persisted `**Started (epoch):**` field and an honest caveat — because the check runs at turn boundaries, real overshoot can be up to one turn's duration past the budget. We document that rather than pretend to second precision.
- **The PM-vs-chair adjudication, and the deciding fact.** The PM ruled NO on `max_wall_seconds` on solid grounds (below). The chair overruled the bottom line on a fact the PM lacked: the PM had read the worktree's PLAN.md at HEAD, which did **not** carry the user's uncommitted edit. That edit promoted `max_wall_seconds` to a live "implement" line *while striking its two siblings*. The standing memory gated wall-clock's return on "evidence that a user shows up wanting it"; a deliberate user edit that adds this line and cuts the others around it **is** that evidence. The PM was reasoning correctly from stale inputs.
- **Absent-by-default — honoring the dissent in the design.** Rather than overrule-and-proceed, the chair adopted the PM's discipline into the feature's shape. The trigger ships with no config line written; absence is the off state. This dodges the Phase-3 `max_tokens` mistake of a visible-but-inert knob that reads as a feature while doing nothing.
- **Relocated considered-and-declined framing.** The PM proposed rendering `max_wall_seconds` as considered-and-declined. Since the user actually implemented it, that framing was relocated onto **per-seat model routing**, which the user did cut — so the declined-with-reasons treatment lands on the thing genuinely declined.
- **The two flagged-item rulings.** Per-seat model routing: **declined**, with the dangling §3/§8 forward-pointers fixed so nothing points at a feature that won't exist. Recreate-safety: **deferred**, left as a clean Phase-4 §9 entry.
